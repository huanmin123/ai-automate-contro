# 文档架构示例

本项目的文档分成两层：

- `handbook/`: 面向使用者的组件手册，解决“这个动作怎么写”
- `docs/`: 面向项目本身的设计与维护文档，解决“这套框架为什么这样设计、未来怎么演进”

另外还有两类 plan 资产：

- `plans/`: 默认示例和发行包 plan 工作区
- `test-plans/`: 源码开发和回归测试夹具

## 使用建议

- 日常写计划时，先看 `handbook/`
- 调整架构或新增能力时，先看 `docs/architecture/`
- 需要排期和推进建设时，先看 `docs/plans/`
