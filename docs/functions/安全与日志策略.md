# 本地调试原文日志策略

## 定位

本项目是本机脚本自动化内核，不是多租户服务端程序。日志、报告、失败现场、AI 会话归档和 plan 产物的首要目标是排查问题，因此默认保留原文。

## 强制规则

- 不做自动脱敏、遮罩、替换、隐藏或摘要化。
- 不因字段名包含账号、密码、token、api_key、Cookie、storage state、验证码等语义而拒绝写入。
- 不把明文凭据自动改写为环境变量、模板引用或占位符。
- 不因为命令文本包含删除、格式化、进程终止、`git reset --hard` 等关键词而拦截执行。
- HTTP 请求头、请求体、响应体、表单、上传文件路径、命令 stdout/stderr、页面 HTML、DOM、截图和图片 data URL 都按原文保留。
- 本机输入文件、上传文件、storage state、HAR、命令 stdin 和命令工作目录允许使用绝对路径、共享盘、外部工作目录或越出 plan 包的相对路径；包内 `resources/` 只是可复现性推荐，不作为拒绝条件。

## 保留的非脱敏约束

- 输出动作仍按 plan 产物规则写入当前 plan 包 `output/` 的对应分区，方便复现和清理。
- JSON schema、字段互斥、必填字段、文件是否存在和输出路径结构等确定性校验继续保留。
- 外部网站验证码、二次验证和人工确认按页面正常流程处理，不实现破解或绕过。

## 维护要求

新增功能、工具、自检或文档时，不要重新引入敏感字段掩码、明文凭据拒写、图片 base64 替换、服务密钥隐藏、危险命令关键词拦截或本机输入路径安全拦截。确需隐藏内容时必须由用户明确要求，并写清这是用户指定的例外。
