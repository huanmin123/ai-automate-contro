# 桌面视觉定位设计

## 定位

`desktop_vision` 是桌面线的视觉兜底 action。当前支持 `type=locate_image` 和 `type=locate_text`，从屏幕、窗口、控件截图或已有图片中定位模板图或 OCR 文本，并输出可复用的屏幕全局 `bounds/point`、source 局部 `local_bounds/local_point`、坐标诊断和证据文件。

实际点击、滚动、拖拽仍交给 `desktop_input target=bounds_center` 或控件级 `desktop_element`。`desktop_vision` 只算取证和定位，不直接执行操作。

## 使用优先级

1. 优先使用应用 API、CLI、导入导出能力。
2. Web 内容使用浏览器执行线和 Playwright selector。
3. 桌面 App 优先使用 Window Query + Element Locator。
4. 控件树不可见、自绘界面、画布、图标按钮、图片按钮或可见短文本无法通过控件树读取时，才使用 `desktop_vision`。
5. 视觉结果不稳定时使用 `manual_confirm`，不要凭猜测坐标继续操作。

## locate_image

```json
{
  "action": "desktop_vision",
  "type": "locate_image",
  "desktop": "desk",
  "template_path": "resources/save-button.png",
  "source_target": "window",
  "title_contains": "Demo",
  "window_match_index": 0,
  "threshold": 0.88,
  "match_index": 0,
  "max_matches": 10,
  "timeout_ms": 3000,
  "interval_ms": 250,
  "path": "save-button-match.json",
  "save_as": "save_button"
}
```

## locate_text

```json
{
  "action": "desktop_vision",
  "type": "locate_text",
  "desktop": "desk",
  "source_path": "output/desktop-screenshots/screen.png",
  "text_contains": "Order Total",
  "language": "eng",
  "provider": "tesseract",
  "min_confidence": 0.6,
  "match_index": 0,
  "max_matches": 10,
  "path": "order-total-ocr.json",
  "save_as": "order_total"
}
```

## 参数

| 字段 | 作用 |
| --- | --- |
| `type` | `locate_image` 或 `locate_text` |
| `desktop` | 桌面 session 名 |
| `template_path` | `locate_image` 使用的模板图，建议放在当前 plan 包 `resources/` |
| `text` / `text_contains` / `text_regex` | `locate_text` 的文字匹配条件，至少一个，建议只写一个 |
| `language` | OCR 语言，默认 `eng`，可用 `eng+chi_sim` 等 Tesseract 语言组合 |
| `provider` | OCR provider，当前 `auto` 或 `tesseract` |
| `min_confidence` | OCR 命中最低置信度，默认 `0.6`，取值 `0..1` |
| `case_sensitive` | OCR 文本匹配是否区分大小写，默认 `false` |
| `source_path` | 可选，已有截图或图片；不能和 `source_target` 同时使用 |
| `source_target` | 可选，`screen`、`window`、`element`；省略且没有 `source_path` 时抓取当前桌面截图 |
| Window Query | `source_target=window/element` 时必填至少一种窗口定位字段 |
| `window_match_index` | 仅 `source_target=window/element` 使用，选择第几个窗口候选，默认 `0` |
| Element Locator | `source_target=element` 时必填至少一种控件定位字段 |
| `region` | 可选搜索区域；使用 `source_path` 时相对图片，`source_target=screen` 时相对屏幕，`source_target=window/element` 时相对窗口或控件截图 |
| `state` | 仅 `source_target=element` 使用，`exists/enabled/disabled/focused`，默认 `exists` |
| `threshold` | 图像匹配最低分，默认 `0.85`，取值 `0..1` |
| `match_index` | 多个命中时选择第几个，默认 `0`；不作为窗口候选索引 |
| `max_matches` | 最多保留的候选数量，默认 `10` |
| `timeout_ms` / `interval_ms` | 等待视觉目标出现 |
| `max_depth` / `max_elements` | `source_target=element` 的控件树遍历限制 |
| `path` | 写入 `output/desktop-vision/` 的 JSON 文件 |
| `save_as` | 保存 payload 到变量 |

## Payload

共同字段：

```json
{
  "ok": true,
  "action": "desktop_vision",
  "type": "locate_image",
  "desktop": "desk",
  "coordinate_space": {"origin": "screen", "unit": "logical_px", "scale": null},
  "coordinate_diagnostics": {
    "source_bounds": {"x": 100, "y": 80, "width": 900, "height": 640},
    "source_size": {"width": 900, "height": 640},
    "global_origin": {"x": 100, "y": 80},
    "local_to_global_offset": {"x": 100, "y": 80},
    "scale": null,
    "warnings": ["scale_unknown"]
  },
  "source_target": "window",
  "source_bounds": {"x": 100, "y": 80, "width": 900, "height": 640},
  "matches": [],
  "match": {
    "index": 0,
    "score": 0.92,
    "confidence": 0.92,
    "bounds": {"x": 420, "y": 180, "width": 96, "height": 32},
    "local_bounds": {"x": 320, "y": 100, "width": 96, "height": 32},
    "point": {"x": 468, "y": 196},
    "local_point": {"x": 368, "y": 116}
  },
  "artifacts": {
    "source_path": "desktop-vision/save-button-source.png",
    "crop_path": "desktop-vision/save-button-crop.png",
    "annotation_path": "desktop-vision/save-button-annotated.png"
  },
  "diagnostics": {}
}
```

`locate_text` 额外包含：

- `raw_text`: OCR 原文。
- `ocr_blocks`: OCR 行块，包含文本、置信度、全局 bounds 和局部 bounds。
- `match.text`: 命中文本。
- `match.confidence`: OCR 置信度。
- `language`、`provider`、`min_confidence`。
- `diagnostics.method/engine_version/word_count/line_count/candidate_count`。

## 能力矩阵

`capability_matrix.capabilities.vision` 暴露可用视觉能力，依赖放在顶层 `capability_matrix.dependencies`：

```json
{
  "capabilities": {
    "vision": {
      "image_locator": true,
      "template_matching": true,
      "ocr": true
    }
  },
  "dependencies": {
    "Pillow.ImageGrab": true,
    "opencv-python": true,
    "tesseract": true,
    "tessdata.eng": true,
    "tessdata.chi_sim": false,
    "pyautogui": true,
    "pyperclip": true
  }
}
```

缺少截图或 OpenCV runtime 时，AI 不应生成 `locate_image` plan；缺 Tesseract 或对应语言包时不要生成 `locate_text` plan。

## 坐标诊断

当前已统一输出：

- `coordinate_space`: `origin`、`unit`、`scale`。
- `source_bounds`: source 在屏幕全局逻辑像素中的位置和尺寸。
- `match.local_bounds/local_point`: source 内局部坐标。
- `match.bounds/point`: 屏幕全局逻辑像素。
- `coordinate_diagnostics.local_to_global_offset`: 从局部坐标换算到屏幕坐标的偏移。
- `coordinate_diagnostics.warnings`: 例如 `scale_unknown`、`negative_source_origin`。

完整 DPI/Retina/多显示器校准仍由后续 `CoordinateMapper` 承接；当前 Windows 回归先验证结构和 `global = local + source_bounds.xy`。

## 风险

- 模板匹配受主题、缩放、语言、hover、disabled 状态和抗锯齿影响。
- OCR 受字体大小、对比度、语言包和中英文混排影响。
- macOS 截图需要 Screen Recording 权限，控件操作还需要 Accessibility。
- 视觉定位失败必须保留原图、候选、分数、OCR 原文和诊断 JSON，不能静默退化成猜坐标。
