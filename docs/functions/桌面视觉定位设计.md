# 桌面视觉定位设计

## 定位

`desktop_vision` 是桌面线的视觉兜底 action。当前已支持 `type=locate_image`；`type=locate_text` 仍是后续 OCR 能力。它只负责从桌面截图或已有图片中定位模板图，并输出可复用的 `bounds`、`point` 和证据文件；实际点击、滚动、拖拽仍交给现有 `desktop_input target=bounds_center` 或控件级 `desktop_element`。

AI 创建可运行 plan 时应优先使用已落地的 `desktop_window`、`desktop_element`、`desktop_capture`、`desktop_assert` 和 `desktop_input`。控件树不可用、自绘 UI、图标按钮或已有截图定位时，可以使用 `desktop_vision type=locate_image` 取证。

## 使用优先级

1. 优先使用应用 API、CLI、导入导出能力。
2. Web 内容使用浏览器执行线和 Playwright selector。
3. 桌面 App 优先使用 Window Query + Element Locator。
4. 控件树不可见、自绘界面、画布、图标按钮或图片按钮时，才使用 `desktop_vision`。
5. 视觉结果不稳定时使用 `manual_confirm`，不要凭猜测坐标继续操作。

## Action

图像模板定位：

```json
{
  "action": "desktop_vision",
  "type": "locate_image",
  "desktop": "desk",
  "template_path": "resources/save-button.png",
  "region": {"x": 0, "y": 0, "width": 1200, "height": 800},
  "threshold": 0.88,
  "match_index": 0,
  "max_matches": 10,
  "timeout_ms": 3000,
  "interval_ms": 250,
  "path": "save-button-match.json",
  "save_as": "save_button"
}
```

已有截图中定位：

```json
{
  "action": "desktop_vision",
  "type": "locate_image",
  "desktop": "desk",
  "template_path": "resources/save-button.png",
  "source_path": "output/desktop-screenshots/screen.png",
  "threshold": 0.88,
  "path": "save-button-match.json",
  "save_as": "save_button"
}
```

OCR 文字定位仍是草案：

```json
{
  "action": "desktop_vision",
  "type": "locate_text",
  "desktop": "desk",
  "text_contains": "保存",
  "region": {"x": 0, "y": 0, "width": 1200, "height": 800},
  "language": "chi_sim+eng",
  "confidence": 0.7,
  "path": "save-text-ocr.json",
  "save_as": "save_text"
}
```

消费视觉结果：

```json
{
  "action": "desktop_input",
  "type": "click",
  "desktop": "desk",
  "target": "bounds_center",
  "bounds": "{{save_button.match.bounds}}"
}
```

## 参数

| 字段 | 作用 |
| --- | --- |
| `type` | 当前支持 `locate_image`；`locate_text` 规划中 |
| `desktop` | 桌面 session 名 |
| `template_path` | `locate_image` 使用的模板图，建议放在当前 plan 包 `resources/` |
| `source_path` | 可选，已有截图或图片；未传时抓取当前桌面截图 |
| `text` / `text_contains` / `text_regex` | `locate_text` 的文字匹配条件，至少一个 |
| `region` | 可选截图区域，使用屏幕逻辑像素；未传时使用全屏 |
| `threshold` | 图像匹配最低分，默认建议不低于 `0.85` |
| `confidence` | OCR 文本最低置信度，默认建议不低于 `0.70` |
| `match_index` | 多个匹配时选择第几个，默认 `0` |
| `max_matches` | 最多保留的候选数量，默认 `10` |
| `timeout_ms` / `interval_ms` | 等待视觉目标出现 |
| `path` | 写入 `output/desktop-vision/` 的 JSON 文件 |
| `save_as` | 保存 payload 到变量 |

## Payload

```json
{
  "ok": true,
  "action": "desktop_vision",
  "type": "locate_image",
  "desktop": "desk",
  "coordinate_space": {"origin": "screen", "unit": "logical_px", "scale": null},
  "region": {"x": 0, "y": 0, "width": 1200, "height": 800},
  "matches": [],
  "match": {
    "index": 0,
    "score": 0.92,
    "confidence": 0.92,
    "bounds": {"x": 420, "y": 180, "width": 96, "height": 32},
    "point": {"x": 468, "y": 196}
  },
  "artifacts": {
    "source_path": "desktop-vision/save-button-source.png",
    "crop_path": "desktop-vision/save-button-crop.png",
    "annotation_path": "desktop-vision/save-button-annotated.png"
  },
  "diagnostics": {}
}
```

OCR payload 还应包含 `raw_text`、`ocr_blocks` 和命中文本片段；当前尚未实现 OCR。

## 能力矩阵

`capability_matrix.capabilities.vision` 应暴露图像定位能力，依赖仍放在顶层 `capability_matrix.dependencies`：

```json
{
  "capabilities": {
    "vision": {
      "image_locator": true,
      "template_matching": true,
      "ocr": false
    }
  },
  "dependencies": {
    "Pillow.ImageGrab": true,
    "opencv-python": true,
    "pyautogui": true,
    "pyperclip": true
  }
}
```

缺少截图或 OpenCV runtime 时，AI 不应生成 `locate_image` plan；缺 OCR runtime 时不要生成 `locate_text` plan。

## 风险

- 坐标必须声明为屏幕逻辑像素；多显示器、负坐标、DPI/Retina 和 RDP 缩放需要单独校准。
- 模板匹配受主题、缩放、语言、hover、disabled 状态和抗锯齿影响。
- OCR 受字体大小、对比度、语言包和中英文混排影响。
- macOS 截图需要 Screen Recording 权限，控件操作还需要 Accessibility。
- 视觉定位失败必须写原图、候选、分数和诊断，不能静默退化成猜坐标。
