# 打包发布指南

项目支持把 CLI 打成可分发目录，并把目录压成 zip，输出统一放到仓库根目录的 `out/`。当前只支持：

- Windows：在 Windows 机器上打 Windows 可执行文件。
- macOS：只支持 Apple Silicon，在 Apple Silicon macOS 上打 macOS 可执行文件。

不支持 Linux，也不支持在 Windows 上打 macOS 包。PyInstaller 不是跨平台交叉编译器；要发布多个平台，就必须分别在对应平台执行打包。

## 打包边界

依赖边界遵循一个原则：如果某个能力在打包后不能通过明确的二次安装命令、系统级关联或外部服务配置补齐，就必须在打包阶段进入 `_internal/`。不能通过 PyInstaller 排除模块、删除 hook 或清理目录来让 Python 代码依赖缺席，再期待用户机器上 `pip install` 修复；exe 启动时使用的是发行目录内的 Python 运行时和 `_internal/` 依赖。

只有具备稳定后补入口和路径约定的外部二进制或资源，才允许不随默认包发布。当前明确支持后补的是 Playwright Chromium：浏览器自动化首次运行前，用户在发行目录执行 `aic install-browser` 或 `cplan install-browser`，安装与当前 Playwright 版本匹配的浏览器二进制。`rg` 这类命令行工具属于系统级 PATH 关联，不进入 `_internal/`。

默认打包产物包含 Python 运行时、项目代码、打包 Python 环境里已经安装的 Python 依赖、`handbook/` 和 `plan.config`，不包含 Playwright Chromium。

需要离线开箱即用时，Windows 打包脚本传 `-BundleBrowser`，macOS 打包脚本传 `--bundle-browser`，才会把 Playwright Chromium 复制进 `_internal/playwright/driver/package/.local-browsers`。这个离线包会明显变大。

打包脚本不会显式排除 Python 代码依赖。默认推荐的打包环境安装 `.[package]`，其中包含打包工具和当前发行包需要随包提供的数据库驱动：PostgreSQL、MySQL、Oracle、SQL Server、Redis 和 MongoDB。桌面控制仍是可选能力；如果要发布支持桌面控制的发行包，就必须先在打包 Python 环境安装 `desktop` extra，再打包。

脚本会显式关闭 UPX 压缩，避免本机 PATH 上存在 UPX 时产生不可压缩 warning 或增加杀软误报概率。

打包脚本每次都会先清理旧的分发目录和 zip；如果旧分发包正在运行，脚本会先尝试结束相关进程，避免打出来的不是最新版。

清理动作只允许发生在仓库 `out/` 和系统临时 PyInstaller 工作目录下，避免路径拼写或同名前缀目录导致误删。Windows 和 macOS 脚本在 PyInstaller 分析前会临时移走包内 `.local-browsers`，避免 `--collect-data playwright` 把本机浏览器缓存误收进轻量包；只有显式选择离线浏览器包时才复制浏览器目录。

打包产物默认不复制 `test-plans/config.json`、`.keygen/` 会话状态、`profiles/` 浏览器状态或 `output/` 运行产物，也不会把旧 `out/ai-automate-contro/plans/config.json` 的本地配置恢复进新的分发目录；这是分发目录来源规则，不是因为真实密钥或敏感字段被拦截。用户运行 exe 时，默认以 exe 所在目录作为运行根目录；也可以通过 `AI_AUTOMATE_PROJECT_ROOT` 指定其他运行根。

发行目录结构：

```text
out/
  ai-automate-contro/
    aic.exe
    _internal/
    handbook/
    plan.config
    plans/
      config.json
  ai-automate-contro-windows-x64.zip
```

`plan.config` 控制 handbook 和 plan 集合位置：

```json
{
  "handbook_path": "handbook",
  "plan_roots": [
    "plans"
  ],
  "default_ai_config_dir": "plans"
}
```

这些路径默认相对于 exe 所在目录。需要真实 AI 服务时，在发行目录的 `plans/config.json` 里添加 `ai_services.default`，或把 `default_ai_config_dir` 指向你自己的配置目录。

发行包使用 PyInstaller onedir 模式：`_internal/` 存放 Python 运行时和依赖，启动时不需要每次解包，比单文件 exe 更适合这个项目。分发时请发送整个 `ai-automate-contro/` 目录或 zip，不要只复制 exe。

AI 终端文本搜索仍然依赖本机安装的 `rg`。目标机器如果要正常使用 AI 终端，需要先安装 `ripgrep`。

## Windows 打包

推荐使用专用打包虚拟环境，避免全局 Python 里无关包、旧 PyInstaller hook 或 PATH 上的工具影响打包结果。第一次准备环境时，在 PowerShell 7 中执行：

```powershell
Set-Location F:\python\ai-automate-contro
python -m venv .\.venv-package
.\.venv-package\Scripts\python.exe -m pip install -U pip
.\.venv-package\Scripts\python.exe -m pip install -e ".[package]"
```

日常正式打包优先使用这条命令：

```powershell
Set-Location F:\python\ai-automate-contro
.\scripts\package.ps1 -PythonPath ".\.venv-package\Scripts\python.exe" -NoProcessCleanup
```

`-PythonPath` 固定使用专用虚拟环境；`-NoProcessCleanup` 避免脚本主动扫描并结束旧打包/分发进程。默认生成轻量包，不下载、不复制 Playwright Chromium。

如果要生成包含 Chromium 的离线包，显式传入 `-BundleBrowser`：

```powershell
Set-Location F:\python\ai-automate-contro
.\scripts\package.ps1 -PythonPath ".\.venv-package\Scripts\python.exe" -NoProcessCleanup -BundleBrowser
```

`-SkipPlaywrightInstall` 只用于 `-BundleBrowser` 场景，表示复用已经安装在当前 Playwright 包目录下的 Chromium，不重新下载。

需要把桌面控制依赖收进包时，先给打包虚拟环境安装 `desktop` extra，再正常打包：

```powershell
.\.venv-package\Scripts\python.exe -m pip install -e ".[package,desktop]"
.\scripts\package.ps1 -PythonPath ".\.venv-package\Scripts\python.exe" -NoProcessCleanup
```

当前 `.[package]` 已经包含发行包需要随包提供的数据库驱动，通常不需要再单独安装 `database-all`；保留 `database-all` 是给源码开发环境一次性安装数据库驱动使用。打包脚本会检查 `psycopg`、`pymysql`、`redis`、`oracledb`、`pyodbc`、`pymongo` 是否已安装；打包虚拟环境必须执行 `pip install -e ".[package]"`。

输出目录和 zip：

```text
out/ai-automate-contro/
out/ai-automate-contro-windows-x64.zip
```

主程序文件名固定为 `aic.exe`，便于启动；zip 文件名仍会跟随当前平台和机器架构变化，例如 `ai-automate-contro-windows-arm64.zip`。

PyInstaller 的工作目录放在系统临时目录，脚本结束后会清理；仓库根目录只保留 `out/` 里的最终产物。

### Windows 蓝屏或系统崩溃排查

打包脚本、Python 或 PyInstaller 是用户态程序，正常失败应该表现为 Python 异常、PyInstaller 报错或脚本退出码非 0；不应该直接导致 Windows 蓝屏。如果打包时出现 `CRITICAL_PROCESS_DIED`、机器重启或系统级崩溃，先按系统/驱动/硬件稳定性问题处理，不要反复直接跑完整打包。

2026-06-21 在 Windows 11 + Python 3.13.2 + PyInstaller 6.21.0 上做过一次分段诊断：标准库、Rich、Textual、Playwright Python 包、OpenAI/LangChain/LangGraph/Pillow 组合小项目都能独立打包并运行；Playwright Chromium 下载解压、主项目 PyInstaller 分析、浏览器目录复制、约 389 MB zip 压缩也都能通过。结论是：当前项目的 Python 依赖组合本身不是稳定蓝屏触发点。更可疑的是当时的全局 Python 环境、旧脚本进程清理、杀软/驱动/VSS/硬件状态或系统稳定性。

遇到类似情况时，先使用诊断输出后缀，不覆盖正式 `out\ai-automate-contro`：

```powershell
Set-Location F:\python\ai-automate-contro
.\scripts\package.ps1 -PythonPath ".\.venv-package\Scripts\python.exe" -OutputSuffix diag -ProbeOnly
.\scripts\package.ps1 -PythonPath ".\.venv-package\Scripts\python.exe" -OutputSuffix diag -NoProcessCleanup -SkipZip -StopAfter DependencyCheck
.\scripts\package.ps1 -PythonPath ".\.venv-package\Scripts\python.exe" -OutputSuffix diag -NoProcessCleanup -SkipZip -StopAfter PlaywrightInstall
.\scripts\package.ps1 -PythonPath ".\.venv-package\Scripts\python.exe" -OutputSuffix diag -NoProcessCleanup -SkipZip -StopAfter PyInstaller
.\scripts\package.ps1 -PythonPath ".\.venv-package\Scripts\python.exe" -OutputSuffix diag -NoProcessCleanup -SkipZip -StopAfter CopyPackage
.\scripts\package.ps1 -PythonPath ".\.venv-package\Scripts\python.exe" -OutputSuffix diag -NoProcessCleanup -SkipZip
.\scripts\package.ps1 -PythonPath ".\.venv-package\Scripts\python.exe" -OutputSuffix diag -NoProcessCleanup
```

每一步都会把检查点写到 `out\package-last-diag.log`。如果某一步导致重启，重启后先看这个日志最后一行，判断停在 `Run PyInstaller`、`CopyPackage` 还是 `Compress package zip`。如果只在 `-BundleBrowser` 下复现，再单独排查 `PlaywrightInstall` 或 `Copy Playwright browsers into package`。不要跳过阶段直接重试完整打包。

诊断包生成后，先跑离线自检：

```powershell
Set-Location F:\python\ai-automate-contro
.\out\ai-automate-contro-diag\cplan.exe self-check cli
.\out\ai-automate-contro-diag\cplan.exe self-check runtime
.\out\ai-automate-contro-diag\cplan.exe validate --file .\out\ai-automate-contro-diag\plans\demo\plan.json
```

轻量诊断包默认没有浏览器，未执行 `aic.exe install-browser` 前不要用 `self-check env` 作为浏览器可用性结论；它会明确提示安装命令和当前浏览器目录。

`out\package-last.log` 或 `out\package-last-diag.log` 是判断打包卡点的第一证据；Windows 事件查看器、`C:\Windows\Minidump\*.dmp` 和 WHEA 事件用于判断系统层问题。磁盘盘符或 D 盘格式化本身通常不会直接影响打包，除非 `TEMP`、pip cache、PyInstaller cache、项目路径或 Playwright 浏览器目录实际指向了该盘。

## macOS Apple Silicon 打包

macOS 使用原生 shell 脚本打包，不需要 PowerShell 7。脚本默认使用 PATH 里的 `python3`，也可以通过 `--python` 或 `PYTHON` 指定解释器；它不会默认创建项目 `.venv`。

```bash
cd /path/to/ai-automate-contro
./scripts/package-macos.sh --install-dependencies --smoke-test
```

默认生成轻量包，不包含 Playwright Chromium。需要离线浏览器包时：

```bash
./scripts/package-macos.sh --install-dependencies --smoke-test --bundle-browser
```

需要打包桌面依赖时，先安装 `desktop` extra，再正常打包。数据库驱动已包含在 `.[package]` 中，不需要额外安装 `database-all`。

```bash
python3 -m pip install -e ".[package,desktop]"
./scripts/package-macos.sh
```

输出文件：

```text
out/ai-automate-contro/
out/ai-automate-contro-macos-arm64.zip
```

分发包包含两个入口：`aic` 是 AI-first 客户端，`cplan` 是无 AI plan 控制 CLI。

如果当前 macOS 不是 Apple Silicon，脚本会直接拒绝打包。

## 使用打包产物

更完整的用户侧启动、配置、AI 终端、图片粘贴和调试流程见 [分发包使用指南](./分发包使用指南.md)。这里保留最小 smoke 命令，方便打包后快速确认产物可用。

建议从发行目录运行，方便真实调试：

```powershell
Set-Location F:\python\ai-automate-contro\out\ai-automate-contro
.\cplan.exe self-check runtime
.\aic.exe install-browser
.\cplan.exe create --path .\plans\demo --automation-type browser
.\cplan.exe validate --file .\plans\demo\plan.json
.\cplan.exe run --file .\plans\demo\plan.json --run-name packaged-smoke
.\aic.exe ai --thread packaged-demo
```

也可以显式指定工作区：

```powershell
$env:AI_AUTOMATE_PROJECT_ROOT = "F:\python\ai-automate-contro"
.\out\ai-automate-contro\cplan.exe self-check runtime
```

`self-check runtime` 只检查 `plan.config`、`handbook/` 和 plan 集合目录。轻量包需要先运行 `aic.exe install-browser`，再运行浏览器 plan 或 `self-check env` 检查 Chromium 可启动性。配置好真实 AI 服务后，再运行 `self-check env` 检查 AI 配置结构。

AI 终端创建 plan 时，如果用户没有明确给目录，会默认使用 `plan.config.plan_roots[0]`。这个发行包默认就是 `plans/`，所以新 plan 会落到发行目录的 `plans/<name>/`。

如果把 exe 发给别人，对方至少需要：

- 一个包含 `plan.json` 的工作区。
- PowerShell 7，用于按文档执行命令。
- `ripgrep`，用于 AI 终端渐进式文本搜索。
- 可访问的 OpenAI-compatible 模型服务配置，如果要使用 AI 终端或专项 AI action。
- Playwright Chromium：轻量包首次运行浏览器 plan 前执行 `aic install-browser`；离线包可通过 `-BundleBrowser` / `--bundle-browser` 预置。
- 可选桌面依赖：默认推荐的打包环境不安装桌面控制 extra；需要 OpenCV、pyautogui、pywinauto 等桌面代码依赖时，先在打包环境安装 `desktop` extra，再重新打包。PostgreSQL、MySQL、Oracle、SQL Server、Redis、MongoDB 数据库驱动属于发行包支持能力，已由 `.[package]` 打入。

## 发布前检查

打包前建议先跑源码侧验证：

```powershell
python -m py_compile .\main.py (Get-ChildItem .\src -Recurse -Filter *.py | ForEach-Object { $_.FullName })
python .\main.py self-check env
python .\main.py self-check ai-terminal
python .\main.py self-check ai-tools
python .\main.py tool check
python .\cplan.py self-check cli
python .\cplan.py self-check runtime
python .\cplan.py self-check browser-components
python .\cplan.py validate --file .\plans\minimal-browser-plan\plan.json
```

打包后建议用 exe 再跑：

```powershell
Set-Location .\out\ai-automate-contro
.\aic.exe install-browser
.\aic.exe self-check env
.\aic.exe self-check ai-stream
.\cplan.exe self-check runtime
.\aic.exe self-check ai-terminal
.\aic.exe self-check textual-client
.\aic.exe self-check ai-tools
.\aic.exe tool check
.\cplan.exe validate --file .\plans\demo\plan.json
.\cplan.exe run --file .\plans\demo\plan.json --run-name packaged-smoke
```

轻量包未安装浏览器前，打包后检查不要运行 `self-check env` 作为必过项；先运行 `.\aic.exe install-browser` 后再检查浏览器能力。离线包使用 `-BundleBrowser` 打包后可以直接跑完整浏览器 smoke。

`out/` 是本地打包产物目录，不应提交到 Git。
