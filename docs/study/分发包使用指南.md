# 分发包使用指南

这份文档面向拿到 `out\ai-automate-contro` 目录或 zip 的使用者。源码开发请看 [快速使用指南](./快速使用指南.md)，打包发布请看 [打包发布指南](./打包发布指南.md)。

## 1. 目录里有什么

Windows 分发包目录默认长这样：

```text
ai-automate-contro/
  aic.exe
  cplan.exe
  _internal/
  handbook/
  plans/
    config.json
  plan.config
```

各部分含义：

- `aic.exe`: AI-first 主程序。
- `cplan.exe`: 无 AI 的 plan 控制程序。
- `_internal/`: 主程序依赖目录，必须和 exe 放在一起，不能只单独复制 exe。
- `handbook/`: 内置动作手册，AI 终端和用户排查时会参考这里。
- `plans/`: 默认 plan 工作区。新建 plan、运行产物和 AI 配置默认都放这里。
- `plans/config.json`: 当前分发目录的集合级配置。需要用真实 AI 服务时，在这里配置。
- `plan.config`: 控制 `handbook` 和 `plans` 的位置。

`plan.config` 默认内容：

```json
{
  "handbook_path": "handbook",
  "plan_roots": [
    "plans"
  ],
  "default_ai_config_dir": "plans"
}
```

相对路径都相对于 exe 所在目录。例如 exe 在 `F:\python\ai-automate-contro\out\ai-automate-contro`，那么默认 plan 会放到：

```text
F:\python\ai-automate-contro\out\ai-automate-contro\plans
```

## 2. 第一次启动

进入分发目录：

```powershell
Set-Location F:\python\ai-automate-contro\out\ai-automate-contro
```

先做运行时自检：

```powershell
.\cplan.exe self-check runtime
.\aic.exe self-check ai-stream
.\aic.exe self-check ai-terminal
.\aic.exe self-check ai-tools
.\aic.exe tool check
```

这些命令不替你检查模型账号是否欠费。真实模型服务返回 503、欠费、协议不兼容或输出不符合 schema 时，程序会直接把错误显示出来，不做手动兼容兜底。

## 3. 必装工具

AI 终端的文本搜索只支持 `ripgrep` 的 `rg` 命令。先检查：

```powershell
rg --version
```

如果没有 `rg`，安装：

```powershell
winget install --id BurntSushi.ripgrep.MSVC -e
```

安装后重新打开 PowerShell 7，再执行：

```powershell
rg --version
```

分发包已经包含 Python 运行时、项目代码、Python 依赖和 Playwright Chromium；正常使用 exe 不需要再安装 Python 包。

## 4. 创建和运行一个最小 Plan

人工创建 plan：

```powershell
.\cplan.exe create --path .\plans\demo --automation-type browser --name "Demo"
```

校验：

```powershell
.\cplan.exe validate --file .\plans\demo\plan.json
```

运行：

```powershell
.\cplan.exe run --file .\plans\demo\plan.json --run-name demo-smoke
```

运行证据和中间产物会写到当前 plan 包的 `output/`：

```text
plans/demo/output/<run>/
```

输入资源默认放入当前 plan 包 `resources/`。如果你把本机文件路径发给 AI，但没有明确要求 plan 长期依赖那个固定路径，AI 会先把文件导入 `resources/`，再在 plan 里使用 `resources/...` 或 `{{resources_file_url}}/...`。plan JSON 内部路径统一使用 `/`，不要使用 Windows 反斜杠。

如果你在需求里明确指定最终文件路径，例如 `Downloads\AI账户.txt`、桌面文件或绝对路径，AI 终端会在 plan 跑通后把整理好的最终结果导出到那个本机路径，不需要你手动复制。

`output/`、`.keygen/`、真实密钥、截图、Cookie、storage state 和日志按本地调试原文优先保留；是否复制、发送或提交由用户决定。

## 5. 配置 AI 服务

如果要使用 AI 终端或普通 plan 里的专项 `ai` action，需要在 `plans/config.json` 配置 OpenAI-compatible 服务：

```json
{
  "ai_services": {
    "default": {
      "base_url": "https://your-openai-compatible-endpoint/v1",
      "model": "your-model",
      "api_key_env": "OPENAI_API_KEY"
    }
  }
}
```

配置后可以检查配置结构：

```powershell
.\aic.exe self-check env
```

注意：项目按 OpenAI-compatible 协议请求模型服务，不会把自定义上下文、事件、thread metadata、附件 metadata 或内部状态字段塞进模型请求体。

## 6. 使用 AI 终端

默认启动程序会进入 Textual AI 客户端：

```powershell
.\aic.exe
```

也可以用下面的命令显式启动同一套 AI 客户端。

启动 AI 终端前，先确认已经按上一节配置了 `plans/config.json`。如果没有配置 `ai_services.default`，程序会提示你编辑哪个配置文件，不会进入 AI 终端。

启动 AI 终端：

```powershell
.\aic.exe ai
```

指定会话 ID 启动：

```powershell
.\aic.exe ai --thread demo-thread
```

脚本或回归测试里可以发起一次性请求并等待结果：

```powershell
.\aic.exe ai --thread demo-thread ask --message "列出当前 plans 下面有哪些 plan 包" --json
.\aic.exe ai --thread demo-thread ask --message "为这个真实网站先探测再创建 plan" --json --events
```

`ai ask` 复用同一套 AI 终端、LangGraph checkpoint、工具调用和上下文压缩机制，只是不进入交互输入行。加 `--events` 后会实时输出 JSONL 事件，方便看到工具调用、plan 进度、等待人工确认和错误。模型服务返回 503、欠费、协议不兼容、schema 不符合或类似 `'str' object has no attribute 'model_dump'` 的错误时，程序会直接显示原始错误，不做本地兼容兜底。需要人工登录、验证码、二次验证或后台菜单操作时，不要用 `ai ask` 承担后续接管；它只能证明流程会停在人工确认点。请在 Textual 客户端里发起任务，并在同一个可见 Playwright 浏览器窗口完成操作后回到客户端回复“继续”。

常用命令：

```text
/status
/new
/sessions 20
/resume 1
```

Textual 客户端不再有 `plan>` 和 `ai>` 模式切换。需要确定性 plan 运行、校验或调试时，直接使用 `.\cplan.exe ...`；AI 工具和 AI 自检继续使用 `.\aic.exe tool ...` 和 `.\aic.exe self-check ...`。

在交互输入行里直接输入 `/` 会弹出命令提示。继续输入前缀会实时过滤候选，例如 `/s` 会保留 `/status`、`/sessions`；需要键盘选择时按 `Up` / `Down` 切换候选，按 `Tab` 或 `Enter` 补全命令。带参数的命令继续按 `Enter` 发送。命令没有重复别名，候选只展示 AI 会话入口；正常使用时仍建议直接用自然语言描述需求。

发送自然语言后，AI 客户端会用灰色输入块回显你的消息，AI 回复直接在普通背景输出，不再带发言人前缀。当前输入区是灰色多行 composer，内容换行或窗口变窄后高度会在有限范围内增长。AI 调用工具时会显示一行工具进度，并在工具完成后原地更新为完成或失败状态，避免刷屏；系统输出、审批和错误会分别显示为独立块。AI 正在回复或运行工具时，你继续发送的普通消息会排队，当前轮完成后继续处理。底部状态条会显示服务、thread、当前 plan、debug workspace、最近 output、运行状态、排队数量、审批状态和待发送图片。模型服务报错会显示 service、model、base_url 和上游错误。AI 返回的 Markdown 会在客户端展示层渲染成标题、加粗、列表和代码块；这只影响屏幕显示，不会改变模型请求、checkpoint、压缩归档或会话原文。需要留存当前屏幕对话时，用 `/export [path]` 导出 Markdown；只保存最近一条 AI 回复时，用 `/copy-last [path]`；清空当前屏幕用 `/clear`。需要排查模型连通性时使用一次性命令 `.\aic.exe ai check --json`。

输入多行内容时，普通 `Enter` 发送；`Shift+Enter`、`Alt+Enter` 或行尾输入 `\` 后按 `Enter` 插入换行；多行粘贴保留换行。单独按 `Esc` 仍用于中断当前 AI/plan，已有队列时会把排队内容作为下一轮介入消息优先处理。

建议使用方式是直接说需求：

```text
帮我在 plans 下面创建一个登录页面巡检 plan，运行后输出 JSON 报告。
```

AI 终端如果创建 plan 且你没有指定目录，默认会放到 `plan.config` 的第一个 `plan_roots`。当前分发包默认就是：

```text
plans/<plan-name>/
```

为真实网站创建 plan 时，AI 会先真实访问页面并读取受限 DOM 摘要；涉及登录、验证码、后台菜单或动态页面时，会继续创建并运行 `open_browser.headed=true` 的探索 plan，把流程先跑通再写最终 plan。需要你介入时，自动化会停在 `manual_confirm`，你应在同一个 Playwright 浏览器窗口里操作，而不是另开自己的浏览器。AI 可以在你授权下正常填写登录字段、点击、提交表单，或输入你提供的一次性验证码/二次验证信息；如果尝试不过、缺少短信/邮箱/设备确认或风险不明确，再交给你处理。不要期待它只凭文字描述猜 selector；用户明确提供并允许用于本地自动化的账号密码可以按需求明文写入 plan 包变量、资源或子计划，不做自动脱敏或拒写。

## 7. 图片怎么给 AI

主路径是粘贴图片后继续打字：

1. 截图或复制图片到剪贴板。
2. 在 AI 终端输入行按 `Ctrl+V`。
3. 如果剪贴板是图片或图片文件，终端会插入类似 `[图片 #1]` 的图片标记。
4. 继续输入文字说明，然后回车发送。

示例输入：

```text
[图片 #1] 看一下这个页面为什么没有提交按钮
```

如果图片已经是文件，可以用文件兜底：

```text
/image .\screenshots\page.png
这张图里登录失败的原因是什么？
```

不需要手动管理附件列表。图片附件会存到当前会话的 `.keygen/ai-terminal-sessions/<thread>/attachments/`，发送给模型前转换成标准 `image_url` 内容；发送成功后清空，模型服务报错时保留，方便修复服务问题后重试。

如果不想发送某张已粘贴图片，把光标放在 `[图片 #n]` 标记后面按 `Backspace`，或放在标记前面按 `Delete`。终端会把整个图片标记一起删除，并同步移除对应附件；剩余图片会自动重编号。光标不会停在 `[图片 #n]` 中间，输入文字时也会自动跳出图片标记，避免把占位符写坏。

## 8. 会话恢复和压缩

AI 终端会把会话状态写到分发目录下的 `.keygen/`：

```text
.keygen/
  ai-terminal-checkpoints.sqlite
  ai-terminal-sessions/
```

查看会话：

```text
/sessions 20
```

恢复会话：

```text
/resume 1
```

或直接用 thread 启动：

```powershell
.\aic.exe ai --thread demo-thread
```

长会话按 128k token 上下文标准估算，在约 64k tokens 时自动压缩，并把完整消息归档到 `.keygen/ai-terminal-sessions/<thread>/compressions/`。模型上下文里只保留摘要和归档路径，不会把完整 `run.log`、`events.jsonl`、`commands.jsonl` 或大型产物全部塞进去。

## 9. 调试失败的 Plan

运行失败后，先看最近输出目录：

```powershell
Get-ChildItem -LiteralPath .\plans\demo\output -Directory | Sort-Object LastWriteTime -Descending | Select-Object -First 5 Name,FullName
```

创建调试工作区：

```powershell
.\cplan.exe debug-prepare --file .\plans\demo\plan.json --name debug-demo
```

调试工作区会放在当前 plan 的 `output/debug/<run>/`，里面有：

```text
source-copy/
injected-plan/
notes.md
report.md
patch.diff
```

AI 或人工只能先改 `injected-plan/`、`notes.md` 或 `report.md`。应用回原始 plan 前必须生成补丁并确认。

## 10. 常见问题

`rg` 不存在：

```powershell
winget install --id BurntSushi.ripgrep.MSVC -e
```

AI 服务报错：

检查 `plans/config.json` 的 `base_url`、`model` 和 `api_key_env` 对应环境变量。如果模型厂商返回欠费、503、协议不兼容或 schema 错误，程序会直接显示原始错误。

AI 终端提示“AI 终端服务未配置：default”：

编辑当前分发目录的 `plans/config.json`，添加：

```json
{
  "ai_services": {
    "default": {
      "base_url": "https://your-openai-compatible-endpoint/v1",
      "model": "your-model",
      "api_key_env": "OPENAI_API_KEY"
    }
  }
}
```

然后验证：

```powershell
.\aic.exe self-check env
.\aic.exe ai
```

不知道当前配置是否正确：

```powershell
.\cplan.exe self-check runtime
.\aic.exe self-check env
.\aic.exe tool check
```

想把 plan 放到别的目录：

编辑 `plan.config` 的 `plan_roots`，或启动前指定运行根：

```powershell
$env:AI_AUTOMATE_PROJECT_ROOT = "D:\my-ai-automate-work"
.\cplan.exe self-check runtime
```

`AI_AUTOMATE_PROJECT_ROOT` 指向的目录里也需要有 `plan.config` 指向的 `handbook/` 和 `plans/`。
