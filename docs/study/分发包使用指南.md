# 分发包使用指南

这份文档面向拿到 `out\ai-automate-contro` 目录或 zip 的使用者。源码开发请看 [快速使用指南](./快速使用指南.md)，打包发布请看 [打包发布指南](./打包发布指南.md)。

## 1. 目录里有什么

Windows 分发包目录默认长这样：

```text
ai-automate-contro/
  ai-automate-contro-windows-x64.exe
  _internal/
  handbook/
  plans/
    config.json
  plan.config
```

各部分含义：

- `ai-automate-contro-windows-x64.exe`: 主程序。
- `_internal/`: 主程序依赖目录，必须和 exe 放在一起，不能只单独复制 exe。
- `handbook/`: 内置动作手册，AI 终端和用户排查时会参考这里。
- `plans/`: 默认 plan 工作区。新建 plan、运行产物和 AI 配置默认都放这里。
- `plans/config.json`: 当前分发目录的集合级配置。需要用真实 AI 服务时，在这里配置。
- `plan.config`: 控制 `handbook` 和 `plans` 的位置。

`plan.config` 默认内容：

```json
{
  "handbook_path": "handbook",
  "plan_roots": [
    "plans"
  ],
  "default_ai_config_dir": "plans"
}
```

相对路径都相对于 exe 所在目录。例如 exe 在 `F:\python\keygen-openai-account\out\ai-automate-contro`，那么默认 plan 会放到：

```text
F:\python\keygen-openai-account\out\ai-automate-contro\plans
```

## 2. 第一次启动

进入分发目录：

```powershell
Set-Location F:\python\keygen-openai-account\out\ai-automate-contro
```

先做运行时自检：

```powershell
.\ai-automate-contro-windows-x64.exe self-check runtime
.\ai-automate-contro-windows-x64.exe self-check ai-stream
.\ai-automate-contro-windows-x64.exe self-check ai-terminal
.\ai-automate-contro-windows-x64.exe self-check ai-tools
.\ai-automate-contro-windows-x64.exe tool check
```

这些命令不替你检查模型账号是否欠费。真实模型服务返回 503、欠费、协议不兼容或输出不符合 schema 时，程序会直接把错误显示出来，不做手动兼容兜底。

## 3. 必装工具

AI 终端的文本搜索只支持 `ripgrep` 的 `rg` 命令。先检查：

```powershell
rg --version
```

如果没有 `rg`，安装：

```powershell
winget install --id BurntSushi.ripgrep.MSVC -e
```

安装后重新打开 PowerShell 7，再执行：

```powershell
rg --version
```

分发包已经包含 Python 运行时、项目代码、Python 依赖和 Playwright Chromium；正常使用 exe 不需要再安装 Python 包。

## 4. 创建和运行一个最小 Plan

人工创建 plan：

```powershell
.\ai-automate-contro-windows-x64.exe plan create --path .\plans\demo --name "Demo"
```

校验：

```powershell
.\ai-automate-contro-windows-x64.exe plan validate --file .\plans\demo\plan.json
```

运行：

```powershell
.\ai-automate-contro-windows-x64.exe plan run --file .\plans\demo\plan.json --run-name demo-smoke
```

运行产物会写到当前 plan 包的 `output/`：

```text
plans/demo/output/<run>/
```

不要把 `output/`、`.keygen/`、真实密钥、截图、Cookie、storage state 或日志发给不可信的人。

## 5. 配置 AI 服务

如果要使用 AI 终端或普通 plan 里的专项 `ai` action，需要在 `plans/config.json` 配置 OpenAI-compatible 服务：

```json
{
  "ai_services": {
    "default": {
      "base_url": "https://your-openai-compatible-endpoint/v1",
      "model": "your-model",
      "api_key": "sk-your-key"
    }
  }
}
```

配置后可以检查配置结构：

```powershell
.\ai-automate-contro-windows-x64.exe self-check env
```

注意：项目按 OpenAI-compatible 协议请求模型服务，不会把自定义上下文、事件、thread metadata、附件 metadata 或内部状态字段塞进模型请求体。

## 6. 使用 AI 终端

默认启动程序会进入统一终端的 `plan>` 管理模式，不会立刻读取或初始化 AI 服务：

```powershell
.\ai-automate-contro-windows-x64.exe
```

在 `plan>` 输入 `ai` 后才会进入 `ai>` 并加载 AI 配置。也可以用下面的命令直接进入 `ai>`。

启动 AI 终端前，先确认已经按上一节配置了 `plans/config.json`。如果没有配置 `ai_services.default`，程序会提示你编辑哪个配置文件，不会进入 AI 终端。

启动 AI 终端：

```powershell
.\ai-automate-contro-windows-x64.exe ai
```

指定会话 ID 启动：

```powershell
.\ai-automate-contro-windows-x64.exe ai --thread demo-thread
```

脚本或回归测试里可以发起一次性请求并等待结果：

```powershell
.\ai-automate-contro-windows-x64.exe ai --thread demo-thread ask --message "列出当前 plans 下面有哪些 plan 包" --json
```

`ai ask` 复用同一套 AI 终端、LangGraph checkpoint、工具调用和上下文压缩机制，只是不进入交互输入行。模型服务返回 503、欠费、协议不兼容、schema 不符合或类似 `'str' object has no attribute 'model_dump'` 的错误时，程序会直接显示原始错误，不做本地兼容兜底。

常用命令：

```text
/status
/new
/sessions 20
/resume 1
/compress
use .\plans\demo\plan.json
run_context .\plans\demo\output\<run>
```

在 `ai>` 输入 `exit` 或 `back` 会回到 `plan>`；输入 `quit` 会退出整个终端。`plan>` 模式也支持命令提示和斜杠形式，例如 `status` 与 `/status` 等价。

在交互输入行里直接输入 `/` 会弹出命令提示，但不会把第一个候选写进输入框。继续输入前缀会实时过滤候选，例如 `/s` 会只保留 `/sessions`、`/status`；需要键盘选择时再按 `Up` / `Down` 切换候选，按 `Enter` 确认。正常使用时仍建议直接用自然语言描述需求。

发送自然语言后，AI 终端会持续显示 `AI>` 回复，同时保留 `ai>` 输入行。AI 正在回复或运行工具时，你继续发送的普通消息会排队，底部状态会显示正在处理和排队数量；按 `Esc` 后发送的下一条消息会排到队首，并在当前回合到达安全边界后处理。模型服务报错会直接显示错误。默认会把 AI 返回的 Markdown 在终端展示层渲染成标题、加粗、列表和代码块；这只影响屏幕显示，不会改变模型请求、checkpoint、压缩归档或会话原文。需要看原始文本时可输入 `render plain` 或 `/render plain`，恢复渲染用 `render markdown`。

输入多行内容时，使用 `Ctrl+J` 插入换行，或者在支持的终端里使用 `Alt+Enter`。部分终端会把 `Shift+Enter` 作为独立按键发给程序，这种情况下也会插入换行；如果当前终端把 `Shift+Enter` 当成普通回车，AI 终端无法区分它和发送键，请改用 `Ctrl+J`。普通 `Enter` 始终表示发送当前消息。

建议使用方式是直接说需求：

```text
帮我在 plans 下面创建一个登录页面巡检 plan，运行后输出 JSON 报告。
```

AI 终端如果创建 plan 且你没有指定目录，默认会放到 `plan.config` 的第一个 `plan_roots`。当前分发包默认就是：

```text
plans/<plan-name>/
```

为真实网站创建 plan 时，AI 会先真实访问页面并读取受限 DOM 摘要，再写自动化步骤。遇到登录、验证码、人机验证、二次验证或权限页面时，它会要求你先完成验证，或生成带 `manual_confirm` 的人机协作 plan；不要期待它只凭文字描述猜 selector，也不要把真实账号密码直接写进 plan。在 `plan>` 手动运行时，`manual_confirm` 仍用 `continue` / `stop`；在 `ai>` 里由 AI 运行 plan 时，确认提示会显示在对话中，你可以用自然语言说明继续或停止。

## 7. 图片怎么给 AI

主路径是粘贴图片后继续打字：

1. 截图或复制图片到剪贴板。
2. 在 AI 终端输入行按 `Alt+V` 或 `Ctrl+V`。
3. 终端会插入类似 `[图片 #1]` 的高亮图片标记。
4. 继续输入文字说明，然后回车发送。

示例输入：

```text
[图片 #1] 看一下这个页面为什么没有提交按钮
```

如果图片已经是文件，可以用文件兜底：

```text
image .\screenshots\page.png
这张图里登录失败的原因是什么？
```

不需要手动管理附件列表。图片附件会存到当前会话的 `.keygen/ai-terminal-sessions/<thread>/attachments/`，发送给模型前转换成标准 `image_url` 内容；发送成功后清空，模型服务报错时保留，方便修复服务问题后重试。

如果不想发送某张已粘贴图片，把光标放在 `[图片 #n]` 标记后面按 `Backspace`，或放在标记前面按 `Delete`。终端会把整个图片标记一起删除，并同步移除对应附件；剩余图片会自动重编号。光标不会停在 `[图片 #n]` 中间，输入文字时也会自动跳出图片标记，避免把占位符写坏。

## 8. 会话恢复和压缩

AI 终端会把会话状态写到分发目录下的 `.keygen/`：

```text
.keygen/
  ai-terminal-checkpoints.sqlite
  ai-terminal-sessions/
```

查看会话：

```text
/sessions 20
```

恢复会话：

```text
/resume 1
```

或直接用 thread 启动：

```powershell
.\ai-automate-contro-windows-x64.exe ai --thread demo-thread
```

长会话按 128k token 上下文标准估算，在约 64k tokens 时自动压缩，并把完整消息归档到 `.keygen/ai-terminal-sessions/<thread>/compressions/`。模型上下文里只保留摘要和归档路径，不会把完整 `run.log`、`events.jsonl`、`commands.jsonl` 或大型产物全部塞进去。

## 9. 调试失败的 Plan

运行失败后，先看最近输出目录：

```powershell
Get-ChildItem -LiteralPath .\plans\demo\output -Directory | Sort-Object LastWriteTime -Descending | Select-Object -First 5 Name,FullName
```

创建调试工作区：

```powershell
.\ai-automate-contro-windows-x64.exe plan debug-prepare --file .\plans\demo\plan.json --name debug-demo
```

调试工作区会放在当前 plan 的 `output/debug/<run>/`，里面有：

```text
source-copy/
injected-plan/
notes.md
report.md
patch.diff
```

AI 或人工只能先改 `injected-plan/`、`notes.md` 或 `report.md`。应用回原始 plan 前必须生成补丁并确认。

## 10. 常见问题

`rg` 不存在：

```powershell
winget install --id BurntSushi.ripgrep.MSVC -e
```

AI 服务报错：

检查 `plans/config.json` 的 `base_url`、`model` 和 `api_key`。如果模型厂商返回欠费、503、协议不兼容或 schema 错误，程序会直接显示原始错误。

AI 终端提示“AI 终端服务未配置：default”：

编辑当前分发目录的 `plans/config.json`，添加：

```json
{
  "ai_services": {
    "default": {
      "base_url": "https://your-openai-compatible-endpoint/v1",
      "model": "your-model",
      "api_key": "sk-your-key"
    }
  }
}
```

然后验证：

```powershell
.\ai-automate-contro-windows-x64.exe self-check env
.\ai-automate-contro-windows-x64.exe ai
```

不知道当前配置是否正确：

```powershell
.\ai-automate-contro-windows-x64.exe self-check runtime
.\ai-automate-contro-windows-x64.exe self-check env
.\ai-automate-contro-windows-x64.exe tool check
```

想把 plan 放到别的目录：

编辑 `plan.config` 的 `plan_roots`，或启动前指定运行根：

```powershell
$env:AI_AUTOMATE_PROJECT_ROOT = "D:\my-ai-automate-work"
.\ai-automate-contro-windows-x64.exe self-check runtime
```

`AI_AUTOMATE_PROJECT_ROOT` 指向的目录里也需要有 `plan.config` 指向的 `handbook/` 和 `plans/`。
