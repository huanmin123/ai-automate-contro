# 对外快速使用指南

这份文档面向第一次使用项目的人。项目本质上是一个 Python + Playwright 的 JSON plan 自动化内核：你用 `plan.json` 描述浏览器、变量、断言、文件读写、控制流和受控 AI 步骤，执行器按步骤运行并把产物写回当前 plan 包的 `output/`。

如果你拿到的是已经打好的 `out\ai-automate-contro` 目录或 zip，优先看 [分发包使用指南](./分发包使用指南.md)。

AI 能力分两类：

- 普通 plan steps 里只能使用受控专项 `ai` action，例如连通性检查、抽取、分类、转换和摘要。
- 开放式对话能力只存在于 plan 级 AI 终端，用来创建、运行、调试和修复 plan。

## 1. 准备环境

推荐环境是 Windows + PowerShell 7 + Python 3.13。项目依赖声明支持 Python 3.11 及以上。

```powershell
Set-Location F:\python\keygen-openai-account

$PSVersionTable.PSVersion
python --version
```

安装项目依赖：

```powershell
python -m pip install -e .
```

安装 `ripgrep`。AI 终端的文本搜索只支持跨平台 `rg`，不使用 Windows 内置搜索兜底。

```powershell
winget install --id BurntSushi.ripgrep.MSVC -e
rg --version
```

安装 Playwright Chromium：

```powershell
python -m playwright install chromium
```

如果 `rg --version` 失败，先重新打开 PowerShell 7 再验证。仍然失败时，请先安装 `ripgrep`，或让维护者在你确认后帮助执行上面的 `winget` 安装命令。

## 2. 本地自检

首次安装后建议先跑自检：

```powershell
python .\main.py self-check env
python .\main.py self-check ai-stream
python .\main.py self-check ai-terminal
python .\main.py self-check ai-tools
python .\main.py tool check
```

这些命令主要检查本地依赖、工具注册、AI 终端会话、图片附件和流式解析。除真实 plan 运行外，自检不会替你检查模型账号是否欠费，也不会隐藏模型服务返回的 503、协议不兼容或 schema 错误。

## 3. 认识 Plan 包

一个 plan 包代表一个独立自动化需求，主入口固定叫 `plan.json`。

```text
my-plan/
  plan.json
  config.json
  resources/
  sub-plans/
  output/
```

常见目录含义：

- `plan.json`: 最小可执行入口。
- `config.json`: 当前 plan 包的局部配置，优先级高于集合级配置。
- `resources/`: HTML、JSON、CSV、TXT 等输入资源。
- `sub-plans/`: 子计划，只能由同包主计划通过 `run_sub_plan` 调用。
- `output/`: 每次运行的日志、截图、JSON、变量、AI 调试产物和失败现场。

运行产物必须留在当前 plan 包的 `output/`。不要把截图、Cookie、storage state、日志、真实密钥或 `.keygen/` 会话状态提交到 Git。

## 4. 运行示例 Plan

先校验公开示例：

```powershell
python .\main.py plan validate --file .\plans\minimal-browser-plan\plan.json
```

再运行：

```powershell
python .\main.py plan run --file .\plans\minimal-browser-plan\plan.json --run-name demo-minimal-browser
```

运行完成后，到对应 plan 包的 `output/` 查看本次 run 目录。常见文件包括 `state.json`、`report.md`、`result.json`、`run.log` 和 `events.jsonl`。

## 5. 使用管理终端

你也可以进入持续管理终端：

```powershell
python .\main.py
```

常用命令：

```text
use .\plans\minimal-browser-plan\plan.json
validate
run demo-minimal-browser
status
logs 80
events 80
artifacts
report
output
```

管理终端是确定性命令入口，适合手动运行、查看输出、继续人工确认流程和创建调试工作区。

## 6. 配置 AI 服务

AI 服务配置来自 plan 集合或当前 plan 包的 `config.json`。公开示例可以使用 `plans/config.json`，真实测试环境可能使用 `test-plans/config.json`。如果文件中包含临时真实服务和 key，请只保留在本机，不要公开分享。

源码开发仓库没有 `plan.config` 时，默认 plan 集合是 `plans/` 和 `test-plans/`；发行包会带 `plan.config`，默认 plan 集合是 exe 同目录下的 `plans/`。AI 终端创建 plan 时，如果用户没有明确给目录，发行包里默认落到 `plan.config.plan_roots[0]`，源码开发仓库里默认落到 `test-plans/ai-generated/<kebab-name>/`。

示例结构如下：

```json
{
  "ai_services": {
    "default": {
      "base_url": "https://your-openai-compatible-endpoint/v1",
      "model": "your-model",
      "api_key": "sk-your-key"
    }
  }
}
```

项目按 OpenAI-compatible 协议请求模型服务。不会把自定义上下文、事件、thread metadata、附件 metadata 或项目内部字段塞进模型请求体。模型厂商返回欠费、503、协议不兼容或输出不符合 schema 时，项目会直接报告错误，不做手动降级、手动重试或改格式兜底。SDK 或 LangChain 内置的传输重试可以保留。

## 7. 使用专项 AI Action

普通 plan steps 里只允许受控专项 `ai` action，并通过 `type` 表示任务类型：

- `connectivity`
- `extract_data`
- `classify_text`
- `transform_data`
- `summarize_text`

专项 AI 必须有固定输入、固定输出 schema 和固定系统提示词。请求摘要、原始响应、解析结果和 schema 校验信息会写入当前 plan 包的 `output/ai/`，便于排查。

## 8. 使用 AI 终端

AI 终端适合让模型帮你围绕某个 plan 做创建、调试、运行、失败分析和补丁建议。

```powershell
python .\main.py ai
python .\main.py ai --thread demo-thread
```

常用命令：

```text
/status
/new
/sessions 20
/resume 1
/compress handoff-before-long-debug
/cancel
use .\plans\minimal-browser-plan\plan.json
workspace .\test-plans\regression\failure-capture\output\debug\<run>
run_context .\plans\minimal-browser-plan\output\<run>
```

图片输入的主路径是“复制图片后继续说话”。截图或复制图片后，在 AI 终端当前输入行按 `Alt+V`；支持该按键的终端会插入 `[Image #1]`，然后直接接着输入文字：

```text
[Image #1] 帮我看这张页面为什么登录失败
```

如果图片已经保存成文件，也可以用 `image .\screenshots\page.png` 作为兜底。图片附件会先落到 `.keygen/ai-terminal-sessions/<thread>/attachments/`，发送前转换为标准 `image_url` data URL。发送成功后清空 pending 图片；如果模型服务报错，附件会保留，方便你修复服务问题后重试。不要把 base64 图片内容写进源码、plan 或文档。

## 9. 上下文与会话恢复

AI 终端会把 checkpoint 写到 `.keygen/ai-terminal-checkpoints.sqlite`，会话摘要写到 `.keygen/ai-terminal-sessions/index.json`。关闭终端后，可以用同一个 thread 恢复：

```powershell
python .\main.py ai --thread demo-thread
```

也可以进入终端后查询和恢复：

```text
/sessions 20
/resume demo-thread
```

长会话按 128k token 上下文标准估算，在约 64k tokens 时自动压缩，并保留约 32k tokens 的近期上下文。完整消息会归档到：

```text
.keygen/ai-terminal-sessions/<thread>/compressions/
```

模型上下文里只保留摘要和归档路径，不会把完整 `run.log`、`events.jsonl`、`commands.jsonl` 或大型产物一股脑塞进去。需要排查时，AI 终端会先看结构和路径，再用 `rg` 定位关键词，最后读取必要的小范围片段。

## 10. 调试和修复 Plan

AI 调试修复必须先创建隔离调试工作区，再把最小补丁应用回原始 plan：

```powershell
python .\main.py plan debug-create --file .\test-plans\regression\failure-capture\plan.json --name debug-login
```

典型调试工作区结构：

```text
output/debug/<run>/
  source-copy/
  injected-plan/
  notes.md
  report.md
  patch.diff
```

AI 或人工只能先修改 `injected-plan/`、`notes.md` 或 `report.md`。应用到原始 plan 前必须生成补丁并经过用户确认。AI 终端请求应用补丁时会进入 human-in-the-loop 审批，只有你输入 `approve` 才会真正应用；`reject <reason>` 会拒绝。

## 11. 常见问题

`rg` 找不到：

```powershell
winget install --id BurntSushi.ripgrep.MSVC -e
rg --version
```

浏览器启动失败：

```powershell
python -m playwright install chromium
python .\main.py self-check env
```

模型服务报 503、欠费、鉴权失败或 schema 不匹配：

检查你的 `base_url`、`model`、`api_key` 和模型服务状态。项目会直接显示这类错误，不会替你切换模型、伪造协议或吞掉异常。

AI 终端显示 busy：

```text
/status
/cancel
```

`/cancel` 的语义是停止等待并忽略当前回合返回结果，不会强杀已经发出的网络请求。

## 12. 上线前检查

交付前建议至少跑：

```powershell
python -m py_compile .\main.py (Get-ChildItem .\src -Recurse -Filter *.py | ForEach-Object { $_.FullName })
python .\main.py self-check env
python .\main.py self-check ai-terminal
python .\main.py self-check ai-tools
python .\main.py tool check
python .\main.py plan validate --file .\plans\minimal-browser-plan\plan.json
python .\main.py plan run --file .\plans\minimal-browser-plan\plan.json --run-name release-smoke-minimal
git diff --check
git ls-files | rg '(^|/)(output|\.keygen|__pycache__)/|\.pyc$|\.egg-info(/|$)'
```

最后一条命令应该没有输出；如果有输出，说明 Git 里已经跟踪了不应该提交的本地产物，需要先从索引中移除。

需要分发可执行文件时，参考 [打包发布指南](./打包发布指南.md)。当前支持在 Windows 本机打 Windows 发行目录和 zip，在 Apple Silicon macOS 本机打 macOS 发行目录和 zip，统一输出到 `out/`。
