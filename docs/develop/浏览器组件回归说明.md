# 浏览器组件回归说明

浏览器组件改动后，优先运行一键回归：

```powershell
python .\cplan.py self-check browser-components
```

这个自检包含两部分：

- 正向回归矩阵：依次校验并运行 `browser-advanced`、`browser-parameter-coverage`、`browser-backlog`、`browser-observability`。
- 负向参数校验：读取 `test-plans/regression/browser-validation-negative/resources/negative-cases.json`，确认错误参数会被 validator 拒绝，并且命中预期错误信息。

正向回归会额外检查运行产物里的证据，不只看 plan 是否 passed：

- `browser-advanced`: 上下文参数、init script、storage、network mock、HAR 和 trace。
- `browser-parameter-coverage`: cookie roundtrip、event clear 后的 console 事件、HAR 录制。
- `browser-backlog`: HAR 回放、coverage、WebSocket 帧、SSE 事件和 ARIA snapshot。
- `browser-observability`: frame 列表、WebRTC 事件和 Service Worker 事件。

## 何时运行

- 修改 `src/ai_automate_contro/engine/actions/` 下浏览器动作。
- 修改 `src/ai_automate_contro/plans/validation_fields.py` 或 `validation_rules.py`。
- 新增或调整 handbook 中浏览器动作字段。
- 发布前做浏览器组件完整回归。

## 前置条件

确保 Chromium 已安装：

```powershell
python -m playwright install chromium
```

如果只改了单个窄范围动作，可以先运行对应 plan；合并前再运行完整自检。
