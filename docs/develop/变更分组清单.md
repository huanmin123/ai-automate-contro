# 变更分组清单

本清单用于提交前分拣当前工作区改动，避免桌面执行线、AI 终端、handbook 重构和历史文档同步混在一个不可审查的提交里。

## 分组建议

### 1. 桌面执行线 runtime 和 action

范围：

- `src/ai_automate_contro/engine/actions/desktop.py`
- `src/ai_automate_contro/engine/actions/executor.py`
- `src/ai_automate_contro/engine/actions/failure_capture.py`
- `src/ai_automate_contro/engine/desktop/**`
- `src/ai_automate_contro/plans/validation_fields.py`
- `src/ai_automate_contro/plans/validation_rules.py`
- `src/ai_automate_contro/plans/validation_walker.py`
- `src/ai_automate_contro/app/desktop_component_check.py`
- `test-plans/desktop/**`

内容：

- desktop action、backend、capability matrix、截图/控件/输入/视觉定位、失败现场和 validation 规则。
- `desktop_vision source_target=window/element` 已返回屏幕全局 `bounds/point` 和 source 局部 `local_bounds/local_point`。
- `desktop_vision window_match_index` 专门选择窗口候选；`match_index` 只选择图像命中，不能再混用窗口语义。
- `desktop_vision type=locate_text` 已接入 Tesseract OCR、`raw_text/ocr_blocks`、全局/局部坐标、标注证据和依赖感知自检。
- Windows 当前回归覆盖 Notepad、Explorer、临时 WinForms、Open/Save 文件对话框；macOS 只保留平台边界和后续真实机器验证入口。

## 当前提交拆分建议

建议后续提交按下列主题拆分，避免一个提交同时包含 runtime、AI 终端、handbook 和长期文档：

1. `desktop runtime/actions`: `engine/actions/desktop.py`、`engine/actions/executor.py`、`engine/desktop/**`、`plans/validation_*`、`app/desktop_component_check.py`。
2. `ai terminal desktop loop`: `src/ai_automate_contro/ai/**` 中与 `inspect_desktop`、质量门禁、执行线确认、桌面 debug 修复和自检相关文件。
3. `cli and release checks`: `app/cli.py`、`app/parser.py`、`app/release_matrix_check.py`、`app/cli_self_check.py`。
4. `handbook cleanup`: `handbook/actions/browser/**`、`handbook/actions/common/**`、`handbook/actions/desktop/**` 和 handbook 索引。
5. `docs architecture and plans`: `docs/architecture/**`、`docs/functions/**`、`docs/plans/**`、`docs/develop/**`。

## 提交前检查

- handbook 内不能出现源码路径、测试命令、回归说明、内部同步清单或开发环境说明。
- desktop/browser/common action 目录必须隔离；旧平铺 action 路径不能再作为 AI 入口。
- 运行 `python .\cplan.py self-check handbook`，自动检查 handbook action 分类、内部 Markdown 链接、旧路径引用和开发噪音。
- 运行 `python .\cplan.py self-check workspace-clean`，自动检查本地产物、日志、pycache、local/secret 配置和 plan output 没有被 git 跟踪。
- 运行 `python .\cplan.py self-check release-matrix --require-desktop-vision`，统一跑当前机器可执行的提交前矩阵，并强制桌面视觉依赖和窗口/控件 source 视觉回归；定位单项问题用 `--list`、`--only` 和 `--fail-fast`。
- 安装 Tesseract 的环境可额外运行 `python .\cplan.py self-check release-matrix --require-desktop-ocr`，强制 OCR 依赖和英文 `locate_text` 回归。
- 真实 AI 回归单独运行：`python .\cplan.py self-check release-matrix --include-real-ai --api-key-file D:\模型密钥.txt --model gpt-5.5 --only ai_real_execution_line --only ai_real_desktop_loop --fail-fast`。
- `review_plan_quality` 对 desktop plan 必须检查桌面探测证据和 plan 内运行证据。
- `inspect_desktop` 返回结构必须是跨平台字段，不把 Windows-only 字段放成顶层契约。
- 提交前确定性回归至少运行 `compileall`、`tool check`、`handbook`、`workspace-clean`、`ai-tools`、`ai-terminal`、`ai-plan-generation`、`desktop-components --require-vision`、`desktop-real-app`、`ai-desktop-loop`。
- 真实模型回归至少运行 `ai-real-desktop-loop` 和 `ai-real-execution-line`，中转服务不稳定时使用现有 `--max-attempts` 和 `--retry-delay-seconds`。
