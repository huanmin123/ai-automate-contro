# 测试与验证说明

当前主要通过示例计划进行功能验证。

所有回归示例应优先使用本地 HTML 夹具，不依赖真实外部网站。

## 建议验证项

- 基础页面打开与输入
- 变量替换
- 数据源读取
- 条件分支
- 循环执行
- 交互式执行器事件流
- 管理终端命令
- AI 终端 plan 创建、失败分析、补丁确认和报告生成
- 失败现场采集：失败截图、失败 HTML DOM、失败页面状态 JSON 和 DOM 摘要
- AI 调试修复闭环：debug prepare、propose debug fix、run debug plan、生成干净 patch
- 专项 AI 组件 schema 校验和 `output/ai/` 产物约束
- 真实 AI 模型连通性、结构化抽取、分类和流程级回归

## 当前验证方式

- 运行 `test-plans/basic/` 下的基础 plan
- 运行 `test-plans/data-driven/` 下的数据驱动 plan
- 运行 `test-plans/ai/controlled-text/` 下的专项 AI plan，覆盖连通性、结构化抽取和分类
- 运行 `test-plans/ai/text-processing/` 下的专项 AI plan，覆盖文本转换和摘要
- 检查 `output/ai/` 下有请求、响应、解析结果和 schema 校验产物
- 运行 `python .\main.py self-check ai-stream`，确认 chat completions streaming chunk、reasoning chunk 忽略、SDK 对象 chunk 和空流拒绝逻辑不依赖外部服务也能回归
- 运行 `python .\main.py self-check ai-terminal`，确认 AI 终端会话索引、会话摘要、恢复选择器、行首 `/command` 命令流、压缩归档、图片附件 metadata 和模型前 OpenAI-compatible 图片消息形态不依赖真实模型也能回归
- 运行 `python .\main.py self-check browser-components`，确认浏览器组件正向矩阵、运行产物证据和负向参数校验都通过
- 运行 `test-plans/regression/failure-capture/` 下的失败采集 plan，确认 `output/<run>/failure-screenshots/`、`failure-html/`、`failure-page-state/` 都存在
- 调用 `analyze_latest_run_failure`，确认 `dom_summaries` 能返回输入框、按钮、链接等关键元素和 selector 提示
- 调用 `propose_debug_fix`，确认 selector 超时能生成候选；应用候选后 `patch.diff` 只包含真实修复，不包含调试注入步骤
- 对 selector 修复同时验证两类场景：没有 `user_hint` 时只返回候选和歧义原因，不写入 `injected-plan/`；提供明确目标提示时才允许自动写入高置信候选
- 运行 `test-plans/regression/patch-apply-check/` 的 HITL 回归，确认 AI 终端请求应用补丁时会进入 `[WAIT_APPROVAL]`，`/reject` 不修改原始 plan，`/approve` 才应用补丁
- 运行 `python .\main.py tool check`，确认 AI 终端工具注册表、Pydantic schema 和工具描述完全对齐
- 运行 `python .\main.py self-check ai-tools`，确认 LangChain `StructuredTool` 构建、共享 Pydantic schema、工具 invoke 回调、渐进式读取工具和受保护工具 HITL 守卫都通过
- 扫描历史入口和移除组件残留，确认不存在顶层 `main.py` 直接运行 plan 的快捷入口、开放式 AI 组件、历史 suite/workspace 中间层或被跟踪缓存文件
- 检查各 plan 包自己的 `output/` 日志结果

## AI 测试配置

`test-plans/config.json` 当前包含用户提供的临时 OpenAI-compatible 测试服务，用于真实 AI 场景回归。该配置可以显式设置 `"stream": true`、`"response_format": "json_schema"`、`"max_retries"` 等调用参数；如果服务欠费、503、协议不兼容或返回内容不符合 schema，项目直接报告错误，不做自动降级、手动重试或兼容兜底。不要在清理旧 AI 代码或文档时删除这段配置；如果密钥过期或需要更换，由用户更新或明确要求移除。

公开示例 `plans/config.json` 不放真实密钥。
