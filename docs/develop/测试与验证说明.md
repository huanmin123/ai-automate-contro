# 测试与验证说明

当前主要通过示例计划进行功能验证。

所有回归示例应优先使用本地 HTML 夹具，不依赖真实外部网站。

## 建议验证项

- 基础页面打开与输入
- 变量替换
- 数据源读取
- 条件分支
- 循环执行
- 交互式执行器事件流
- `cplan` 命令边界与无 AI plan 控制
- AI 终端 plan 创建、失败分析、补丁确认和报告生成
- 失败现场采集：失败截图、失败 HTML DOM、失败页面状态 JSON 和 DOM 摘要
- 桌面自动化落地后：App 启动、窗口列表、聚焦、关闭/最小化/最大化/还原、控件枚举/定位/读取/点击/写值/触发、表格读取/单元格选择、树读取/展开/折叠/选择、菜单栏和右键上下文菜单路径触发、滚动容器语义滚动、键盘输入、坐标点击/双击/右键/滚动/拖拽、全屏/区域/窗口/控件截图、控件断言、`capability_matrix`、标注截图、权限诊断和桌面失败现场
- AI 调试修复闭环：debug prepare、propose debug fix、run debug plan、生成干净 patch
- 专项 AI 组件 schema 校验和 `output/ai/` 产物约束
- 真实 AI 模型连通性、结构化抽取、分类和流程级回归

## AI 测试配置

`test-plans/config.json` 当前包含用户提供的临时 OpenAI-compatible 测试服务元数据，用于真实 AI 场景回归。该配置可以显式设置 `"stream": true`、`"response_format": "json_schema"`、`"max_retries"`、`"api_key"` 或 `"api_key_env"` 等调用参数；项目按本地调试原文优先处理，不因 inline `api_key` 拒绝写入。如果服务欠费、503、协议不兼容或返回内容不符合 schema，项目直接报告错误，不做自动降级、手动重试或兼容兜底。不要在清理旧 AI 代码或文档时删除这段服务配置；如果密钥过期或需要更换，由用户更新本机配置或明确要求移除。

公开示例 `plans/config.json` 可以按本地调试需要保留真实密钥；工具不因密钥字段做脱敏或阻断。
