# 安装指南

## 推荐环境

项目默认开发环境是 Windows + PowerShell 7 + 全局或用户级 Python。日常开发不要求在项目根目录创建 `.venv`；`.venv/` 只视为某台机器上的本地临时环境，已经由 `.gitignore` 忽略，不应写入文档、plan 或脚本路径。

推荐使用 Python 3.13。项目声明支持 Python 3.11 及以上，真实版本要求以 `pyproject.toml` 为准。

先确认当前终端使用的解释器：

```powershell
$PSVersionTable.PSVersion
Get-Command python
python --version
python -m pip --version
```

如果 IDE 无法识别 `src/` 下导入，优先选择同一个全局 Python 解释器，或把 `src` 标记为 Sources Root。

## 安装项目依赖

```powershell
python -m pip install -e .
winget install --id BurntSushi.ripgrep.MSVC -e
rg --version
python -m playwright install chromium
python .\main.py self-check env
```

建议使用 PowerShell 7 作为默认 shell。

`python -m pip install -e .` 会按 `pyproject.toml` 安装开发依赖，当前关键版本如下：

| 依赖 | 版本要求 |
| --- | --- |
| Python | `>=3.11`，推荐 `3.13` |
| Playwright | `==1.59.0` |
| OpenAI SDK | `>=2.36.0,<3.0.0` |
| jsonschema | `==4.26.0` |
| LangChain | `==1.2.18` |
| LangGraph | `==1.1.10` |
| langgraph-checkpoint-sqlite | `==3.0.3` |
| langchain-openai | `==1.2.1` |
| pydantic | `>=2.13.4` |
| typing_extensions | `>=4.15.0` |
| Pillow | `>=11.0.0` |
| prompt_toolkit | `>=3.0.52,<4.0.0` |
| Rich | `>=13.9.4,<16.0.0` |

如果只需要补装单个缺失依赖，例如 `pydantic`：

```powershell
python -m pip install "pydantic>=2.13.4"
python -c "import pydantic; print(pydantic.__version__)"
```

如果某台机器把 pip 配成了必须使用虚拟环境，先确认配置来源：

```powershell
python -m pip config list -v
```

Windows 开发机建议解除这类本地限制后继续使用全局或用户级 Python。PowerShell 临时覆盖示例：

```powershell
$env:PIP_REQUIRE_VIRTUALENV = "false"
python -m pip install "pydantic>=2.13.4"
```

macOS Homebrew Python 可能同时启用 PEP 668 保护；临时维护时可以装到用户级 site-packages，避免写入项目 `.venv`：

```bash
PIP_REQUIRE_VIRTUALENV=false python3 -m pip install --user --break-system-packages "pydantic>=2.13.4"
```

AI 终端的渐进式文本搜索依赖 `ripgrep` 的 `rg` 命令，项目不使用 Windows 内置搜索做兜底。如果 `rg --version` 失败，先安装 `ripgrep`，或让用户确认后由助手执行上面的全局安装命令。安装后如果当前终端还找不到 `rg`，重新打开 PowerShell 7 再验证。

`python .\main.py self-check env` 会检查 Python、PowerShell 7、项目依赖、`rg`、Playwright Chromium 和本地 AI 配置结构。它不会访问真实模型服务；欠费、503、协议不兼容等真实模型错误会在运行 AI 终端或专项 AI plan 时直接暴露给用户。
