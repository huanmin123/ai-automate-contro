# 问题-0001-严格模式选择器冲突

## 现象

等待登录卡片内输入框时，选择器一次命中了用户名、密码、验证码三个输入框，导致 Playwright 严格模式报错。

## 复现步骤

1. 使用过宽的登录卡片输入框选择器
2. 运行示例计划

## 原因分析

等待动作使用的选择器没有收敛到单一字段，且执行器最初版本对这类场景的处理不够友好。

## 修复方案

- 将示例选择器改为 `input[autocomplete='username']`
- 调整 `wait_for_selector`，允许在不指定 `index` 时等待匹配集合

## 验证结果

示例计划再次执行后恢复正常。

## 预防措施

- 优先使用更稳定、更语义化的选择器
- 不要默认相信长 CSS 路径更准确
