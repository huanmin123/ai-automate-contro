# 问题-0004-模型节点超时与服务能力边界

## 现象

`llm_chat` 节点已经打通 OpenAI 兼容接口，但在当前 Linux 模型网关配置下，请求可能出现长时间等待并最终超时。

## 现阶段判断

- OCR sidecar 链路已验证可用
- `GET /healthz`、`GET /v1/models` 正常
- 实际 `POST /v1/chat/completions` 生成存在超时风险

## 原因分析

这类问题更接近模型运行时能力与超时策略问题，而不是自动化框架动作契约问题。

可能因素包括：

- 当前默认模型响应较慢
- 本地模型机负载较高
- 超时阈值偏保守
- 网关对长生成请求没有做额外流式或异步优化

## 当前处理策略

- 保留 `llm_chat` 与 `llm_extract_json` 节点接口
- 文档中明确标注当前验证状态
- 优先用 OCR 节点推进 AI 层建设

## 后续建议

- 先在模型机侧单独压测 `/v1/chat/completions`
- 根据模型实际时延调大 `timeout_seconds`
- 必要时增加更轻量的默认模型配置
