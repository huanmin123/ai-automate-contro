# 计划-0002-框架第三版建设

> 历史记录：本计划中的 suite 汇总和标签筛选方案已废弃。当前只直接执行 `plan.json`，多个 plan 的批量调度应由外部脚本负责。

## 背景

第二版已经具备长期复用的基础能力，但在“批量执行可观测性”和“按场景选择性回归”方面还不够完善。

## 目标

- 失败时自动截图
- 为每个计划输出统一结果文件
- 为 suite 输出汇总报告（已废弃）
- 支持通过标签筛选项目和场景（已废弃）

## 范围

- 计划结果文件
- suite 汇总文件（已废弃）
- CLI 标签过滤参数（已废弃）
- 相关手册与架构文档同步

## 实施步骤

1. 增强运行结果结构
2. 在失败时保存截图
3. 在 suite 结束时写汇总（已废弃）
4. 在 CLI 和 suite 中加入标签过滤（已废弃）
5. 补示例与文档

## 风险

- 并发写日志时可能互相打架
- 失败截图过多时会增加输出体积

## 验收标准

- 失败计划能够留下截图
- suite 结束后存在 Markdown 和 JSON 汇总（历史目标，当前不再支持）
- 可以通过标签只跑部分项目（历史目标，当前不再支持）

- `docs/functions/核心功能设计.md`
- `docs/develop/运行说明.md`
- `handbook/计划结构.md`
