# 计划-0012-真实数据库回归与本机配置落地

## 背景

SQLite 默认回归已经稳定，`sql`/`redis` common action 也具备真实服务可选自检入口。下一步需要把本机真实中间件资源纳入显式回归：MySQL、PostgreSQL、Redis、Oracle 和 Elasticsearch 都能通过 `database-components --include-real-db` 产生结构化结果。

真实连接信息属于本机配置，不写入可提交示例。公开示例只提供占位和环境变量引用。

## 目标

- 增加可提交的数据库服务配置样例。
- 让 `database-components` 支持配置文件里的环境变量引用。
- 明确 `local/database-services.json` 是本机私有配置。
- 在当前机器上跑通真实 MySQL、PostgreSQL、Redis、Oracle 和 Elasticsearch 只读 smoke；需要写入时显式加 `--allow-writes`。
- 保持默认 release matrix 仍只依赖 SQLite。

## 范围

- 更新 `.gitignore` 和 workspace-clean 检查，覆盖 `local/` 本机配置目录。
- 新增 `test-plans/database/database-services.example.json`。
- 增强数据库自检配置加载和诊断输出。
- 用本机真实配置运行可选真实服务回归。

## 不做

- 不把真实密码写入公开样例。
- 不把真实服务回归加入默认离线矩阵。
- 不在没有用户明确授权时连接生产库。
- 不为真实服务自动创建长期表、索引或测试库。

## 实施步骤

1. 把 `local/` 纳入忽略和 workspace-clean 规则。
2. 新增可提交的 `database-services.example.json`，覆盖 MySQL、PostgreSQL、Redis、Oracle、Elasticsearch。
3. 支持配置值使用 `{"env": "NAME"}`、`{"env": "NAME", "default": "value"}`、`"env:NAME"`、`"$env:NAME"` 或 `"${NAME}"` 引用环境变量。
4. 增强 `database-components` 输出，展示配置路径、环境变量引用和未解析项。
5. 使用本机私有 `local/database-services.json` 跑真实服务 smoke。

## 风险

- 真实服务可能因网络、账号、权限或服务状态波动失败，因此只能显式运行。
- MySQL/Oracle 等驱动可用不代表目标数据库账号拥有写权限；默认只读。
- Redis 写入测试必须使用短 TTL、独立 key 前缀并清理；默认不开写入。

## 验收标准

- `python .\cplan.py self-check database-components` 默认通过。
- `python .\cplan.py self-check database-components --include-real-db --database-config .\local\database-services.json` 在当前机器上通过或对缺失配置返回结构化 skipped；需要临时写入时再加 `--allow-writes`。
- `workspace-clean` 不会把 `local/` 本机配置报为可提交源文件。
- 文档说明本机真实配置、环境变量引用和真实服务回归命令。
