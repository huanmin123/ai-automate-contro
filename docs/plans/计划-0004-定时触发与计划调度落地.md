# 计划-0004-定时触发与计划调度落地

## 目标

一次性落地两类定时能力：

- plan 内运行期触发器：浏览器登录后，在同一个 run 内按间隔执行内联步骤或同包子计划。
- CLI 长期定时计划：通过 `cplan schedule` 管理按时间启动完整 plan run 的配置。

## 范围

- 新增父级控制流 `trigger` action，支持 `trigger.steps` 和 `trigger.path`。
- 新增 `cplan schedule` 子命令。
- 新增 AI 终端 schedule 管理工具。
- 更新 handbook、docs 和测试 plan。
- 运行确定性自检和真实命令验证。

## 关键设计

- `steps` 仍是主执行流，trigger 必须作为可执行父 action 写在 `steps` 中；不再使用顶层 `routines` 或 `triggers` 定义区。
- 无限触发必须显式 `allow_infinite: true`。
- 运行期触发器单线程执行，不并发操作同一 Playwright 页面。
- schedule 配置放在 `schedules.json`，运行状态放在 `.keygen/schedules-state.json`。
- CLI 和 AI 终端共享核心 schedule 模块。
