# 计划-0001-框架第二版建设

> 历史记录：本计划中的 suite / examples / 根 output 方案已废弃。当前架构以 plan 包为最小执行单元，项目测试放在 `test-plans/`，运行产物只允许写入当前 plan 包的 `output/`。

## 背景

第一版已经具备基本 JSON 执行能力，但距离“可长期复用、可多项目测试”的目标还有差距。

## 目标

- 支持多浏览器共享变量
- 支持外部数据源
- 支持条件分支与循环
- 支持顺序与并发套件（已废弃，后续由外部脚本批量扫描 plan 包）
- 支持日志输出
- 建立正式项目文档体系

## 范围

本轮聚焦执行核心和文档体系，不处理插件化与 HTML 报表。

## 实施步骤

1. 扩展执行动作与变量系统
2. 引入 suite 执行模型（已废弃）
3. 建立 `examples/data/`、`examples/suites/`、`output/` 约定（已迁移为 `plans/`、`test-plans/` 和 plan 包内 `output/`）
4. 补齐 `handbook/` 和 `docs/`
5. 通过示例计划验证

## 风险

- 动作快速增长后，`actions.py` 会变大
- 并发模式下的资源冲突仍需继续观察

## 验收标准

- 单计划仍可正常运行
- 套件支持顺序与并发（历史目标，当前不再支持）
- 示例能加载 json/csv/txt 数据
- 文档体系可作为后续维护入口

## 文档同步

- `docs/architecture/架构总览.md`
- `docs/functions/核心功能设计.md`
- `handbook/`

## 后续迭代建议

- 失败自动截图
- 外部批量执行脚本
- plan 包级报告聚合
