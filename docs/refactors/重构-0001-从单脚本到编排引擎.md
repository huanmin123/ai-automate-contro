# 重构-0001-从单脚本到编排引擎

> 历史记录：本记录中的 suite 调度模型已经被后续 plan 包架构替代。当前执行边界是单个 `plan.json`，子计划只能存在于同包 `sub-plans/`。

## 背景

项目起点只是一个空仓库和一个示例 `main.py`，无法承载长期自动化需求。

## 旧结构问题

- 没有执行引擎
- 没有计划编排能力
- 没有文档体系
- 没有多项目组织方式

## 新结构设计

- 引入 JSON 计划执行模型
- 引入 suite 调度模型（已废弃）
- 引入 handbook 和 docs 双文档体系
- 引入 output 日志目录

## 风险

- 动作组件增长后需要继续模块化
- 多 plan 批量调度需要外部脚本承接

## 验证

- 单计划示例可运行
- plan 包模型可解析

## 复盘

这次重构完成了从“脚本”到“框架雏形”的跨越。后续架构已经收敛为 plan 包模型，避免 suite、workspace 和 plan 之间的边界混淆。
