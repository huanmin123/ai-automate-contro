# 重构-0001-从单脚本到编排引擎

## 背景

项目起点只是一个空仓库和一个示例 `main.py`，无法承载长期自动化需求。

## 旧结构问题

- 没有执行引擎
- 没有计划编排能力
- 没有文档体系
- 没有多项目组织方式

## 新结构设计

- 引入 JSON 计划执行模型
- 引入 suite 调度模型
- 引入 handbook 和 docs 双文档体系
- 引入 output 日志目录

## 风险

- 动作组件增长后需要继续模块化
- 并发套件仍需要更多真实场景验证

## 验证

- 单计划示例可运行
- 套件模型可解析

## 复盘

这次重构完成了从“脚本”到“框架雏形”的跨越，但后续仍要继续做测试报告、动作拆分和更稳定的场景抽象。
