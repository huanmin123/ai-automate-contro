# 架构示例

一个典型场景：

1. `<plan-root>/<requirement>/plan.json` 定义一个独立需求，`plan-root` 来自当前运行根的 `plan.config.plan_roots`
2. `plan.json` 可以引用同包内的 `sub-plans/*-plan.json`
3. `resources/` 存放这个需求独占的夹具和数据
4. `output/` 存放这个需求的运行结果
5. `plan` 内部通过动作组件驱动浏览器

这意味着“需求包”和“页面动作”已经分层：

- `plan` 负责一个完整需求
- `sub-plan` 负责需求内部可读性拆分
- `action` 负责最小动作原子化
