# 功能开发指导

新增功能时，优先判断它属于哪一层：

- 如果是一个新的页面原子动作，新增 `action`
- 如果是多个动作的组合能力，优先考虑通过 `plan` 编排完成
- 如果是多个计划的组织方式，优先考虑扩展 `suite`

## 新增动作的标准流程

1. 在 `actions.py` 新增 `_action_xxx`
2. 如有条件逻辑，补充 `conditions.py`
3. 在 `handbook/` 新增对应组件文档
4. 在合适的 `examples/scenarios/` 子目录新增一个最小示例
5. 在 `docs/functions/核心功能设计.md` 记录能力边界

## 不建议的做法

- 把特定站点逻辑硬编码进执行器核心
- 用超长脆弱选择器替代更稳定的选择器策略
- 只加代码不补文档
