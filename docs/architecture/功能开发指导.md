# 功能开发指导

新增功能时，优先判断它属于哪一层：

- 如果是一个新的页面原子动作，新增 `action`
- 如果是多个动作的组合能力，优先考虑通过 `plan` 编排完成
- 如果是复杂流程，优先拆到同一个 plan 包的 `sub-plans/`

## 新增动作的标准流程

1. 在 `src/ai_automate_contro/engine/actions/` 下选择合适行为模块新增动作；如果要新建模块，同步 action 注册出口和 validator 支持集合
2. 如有条件逻辑，补充 `src/ai_automate_contro/engine/conditions.py`
3. 在 `handbook/actions/` 下的合适分类新增对应组件文档，并确认 `handbook/README.md` 的按需地图能指到它
4. 在合适的 `test-plans/` 子目录新增或更新一个最小 plan 包
5. 在 `docs/functions/核心功能设计.md` 记录能力边界

## 不建议的做法

- 把特定站点逻辑硬编码进执行器核心
- 用超长脆弱选择器替代更稳定的选择器策略
- 只加代码不补文档
