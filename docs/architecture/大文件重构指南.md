# 大文件重构指南

当前 `actions.py` 会随着能力增长继续变大，后续应考虑拆分。

## 推荐拆分方向

- `actions/browser_actions.py`
- `actions/navigation_actions.py`
- `actions/input_actions.py`
- `actions/extract_actions.py`
- `actions/control_flow_actions.py`
- `actions/data_actions.py`

## 什么时候该拆

- 单文件超过 500 行并开始难以导航
- 新人需要较长时间才能定位动作实现
- 修改一个能力时，容易误伤别的动作

## 拆分原则

- 保持动作名不变
- 对外 JSON 契约不变
- 文档链接不变或平滑迁移
