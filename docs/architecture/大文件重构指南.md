# 模块重构指南

动作执行层已从旧单文件结构迁移为 `src/ai_automate_contro/engine/actions/` 包。后续新增能力时，应按职责边界维护内聚模块，避免把所有 action 重新堆回 `executor.py`，也避免为了文件大小继续过度切碎。

## 推荐拆分方向

- `engine/actions/browser_flow.py`: 浏览器、页面、导航、元素和等待相关动作
- `engine/actions/files.py`: 读写文件动作
- `engine/actions/assertions.py`: 断言动作
- `engine/actions/failure_capture.py`: 失败截图、HTML 和页面状态捕获
- `engine/actions/basic.py`: 变量、读写、断言、打印、人工确认和 sleep 等基础动作入口
- `engine/actions/control_flow.py`: `run_sub_plan`、`if`、`foreach`、`retry`
- `engine/actions/extraction.py`: 页面内容抽取和表格抽取
- `engine/actions/ai_task.py`: 受控专项 `ai` action
- `engine/actions/executor.py`: step 生命周期、handler 分发、失败捕获和共享运行上下文

## 什么时候该拆

- 单文件超过约 1000 行，并且已经难以导航或 review
- 一个文件承担多个生命周期明显不同的职责
- 修改一个能力时，容易误伤别的动作或共享路径
- 某块逻辑需要独立回归、独立替换或隔离高风险依赖

## 拆分原则

- 保持动作名不变，并同步 action 注册出口和 plan validator 支持集合
- 对外 JSON 契约不变
- 文档链接不变或平滑迁移
- 不用“行数少一点”作为拆分理由；1000 行以内且职责清晰的模块优先保持内聚
- 拆分后的模块名必须表达行为域，不能形成只有一两个函数的无意义碎片
