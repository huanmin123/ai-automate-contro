# 多项目与标签执行

这份文档说明怎么组织多个项目、多个场景，以及怎么按标签筛选执行。

## suite 是什么

如果 `plan` 是一个单场景执行文件，那么 `suite` 就是多个计划的调度文件。

你可以把 3 个项目、3 个测试场景，或者 3 组不同数据的回归任务放到一个 suite 里统一运行。

## 两种执行模式

### 顺序执行

```json
{
  "name": "sequential-projects",
  "mode": "sequential",
  "plans": []
}
```

适合调试和稳定性优先的场景。

### 并发执行

```json
{
  "name": "parallel-projects",
  "mode": "parallel",
  "max_workers": 3,
  "plans": []
}
```

适合项目彼此独立、机器资源足够的场景。

## 给计划打标签

```json
{
  "name": "smoke-login-a",
  "path": "../../scenarios/basic/fill_system_account.example.json",
  "tags": ["smoke", "login", "project-a"]
}
```

## 在 suite 文件里预设标签过滤

```json
{
  "name": "tagged-projects",
  "mode": "sequential",
  "include_tags": ["smoke"],
  "exclude_tags": ["failure-demo"],
  "tag_mode": "any",
  "plans": []
}
```

## 运行时临时筛选标签

```powershell
python .\main.py --file .\examples\suites\smoke\tagged_projects.example.json --include-tags smoke,login --tag-mode all
```

含义：

- `--include-tags smoke,login`: 只保留带这些标签的项目
- `--tag-mode all`: 必须同时包含 `smoke` 和 `login`
- `--tag-mode any`: 只要命中任意一个标签即可

## 输出结果

suite 执行后会生成：

- `suite-summary.json`
- `suite-summary.md`

如果某个子计划失败，还会在它自己的输出目录里生成：

- `result.json`
- `failure-screenshots/`

补充说明：

- `handbook/` 负责教学
- `examples/` 负责提供可运行资产
- 不建议通过直接阅读 `examples/` 里的 JSON 来代替手册
