# 第一个计划示例

这个示例演示最常见的一条链路：打开浏览器，进入页面，等待输入框出现，然后填入邮箱。

## 示例 JSON

```json
{
  "variables": {
    "email": "test@example.com"
  },
  "steps": [
    {
      "action": "open_browser",
      "name": "login",
      "headed": true,
      "slow_mo_ms": 200
    },
    {
      "action": "goto",
      "browser": "login",
      "url": "https://aijh.huanmin.top/"
    },
    {
      "action": "wait_for_selector",
      "browser": "login",
      "selector": "input[autocomplete='username']"
    },
    {
      "action": "fill",
      "browser": "login",
      "selector": "input[autocomplete='username']",
      "value": "{{email}}"
    },
    {
      "action": "wait",
      "browser": "login",
      "seconds": 5
    }
  ]
}
```

## 每一步在做什么

- `open_browser`: 创建一个名为 `login` 的浏览器会话
- `goto`: 打开目标页面
- `wait_for_selector`: 等待用户名输入框真正出现在页面上
- `fill`: 把变量里的邮箱填进去
- `wait`: 暂停 5 秒，方便观察页面结果

## 对应示例文件

- `examples/scenarios/basic/fill_system_account.example.json`

## 适合什么时候用

- 你要验证某个选择器能不能定位到目标元素
- 你要快速试跑一个输入框填充流程
- 你刚开始接触这套 JSON 编排方式

真正学习时以本手册说明为准，不要反过来把 JSON 示例文件当教程读物。
