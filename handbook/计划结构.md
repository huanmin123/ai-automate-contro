# 计划结构

一个计划文件就是一个标准 JSON，对应两个顶层字段：`variables` 和 `steps`。

## 顶层字段

- `variables`: 全局变量字典，供后续步骤通过 `{{变量名}}` 引用
- `steps`: 按顺序执行的动作数组

## 最小结构

```json
{
  "variables": {},
  "steps": []
}
```

## 带变量的常见结构

```json
{
  "variables": {
    "email": "test@example.com",
    "target_url": "https://aijh.huanmin.top/"
  },
  "steps": [
    {
      "action": "open_browser",
      "name": "main",
      "headed": true
    },
    {
      "action": "goto",
      "browser": "main",
      "url": "{{target_url}}"
    },
    {
      "action": "fill",
      "browser": "main",
      "selector": "input[autocomplete='username']",
      "value": "{{email}}"
    }
  ]
}
```

## 约定

- `browser` 字段表示当前步骤作用于哪个浏览器会话。
- `selector` 使用 Playwright 支持的选择器语法。
- `{{email}}` 这种写法会在执行前替换成变量值。
- 步骤按数组顺序执行，前一步失败时，后续步骤不会继续。

## 推荐写法

- 先 `open_browser`，再 `goto`，再做页面交互。
- 在关键节点加 `wait_for_selector`，不要盲目依赖固定睡眠。
- 页面值需要复用时，用 `extract_*` 保存到变量里。
- 人工参与的流程，用 `manual_confirm` 作为暂停点。
